This codelab will walk you through about how to setup your machine for Flutter development. We are going to use a command-line installations using Chocolatey, an installation Manager for Windows.

The installation is only for Windows machine

  • Prepared by: Jumail (jumail@utm.my)
  • for the course: SECJ3623 Mobile Application Programming
  • Date: March 23, 2022

Chocolatey is a third-party tool. So we need to install it first.

Open a Poweshell Terminal as Administrator

  • Right click the Windows Icon. to open Windows menu.
  • Click on Windows Powershell(Admin)

Install Chocolatey

  • In the Powershell Terminal, run the following commands:
    Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
  • Test chocolatey installation
    choco --version

# If the chocolatey successfully installed, you should see the version.

Links

  • Chocolatey Installation
  • Find Package on Chocolatey

In this step, we are going to install the following softwares:

  • Visual Studio Code (Code Editor)
  • Git (Source Code Versioning Tool)
  • Flutter (The SDK)
  • Android Studio (To be used only for SDK Manager and AVD Manager)

We are going to install them all at once. This step will take some times. 

choco install vscode git flutter androidstudio -y

Open A New Powershell Session

The above installations may have changed the Environement Variables of your system such as the PATH. To allow the changes to take effect, open a new powershell session.

In this step onwards, you are expected to work on the newly opened Powershell terminal. You can install VS Code Extensions from the VS Code. However, in this step we are going to do that using command-line in powershell terminal.

Extenstions to install

  • Flutter
  • Live Share (for collaborative coding)
  • Error Lens (Show error at coding-time)
  • Pubspec Assist (install package in flutter project pubspec.yaml)
  • Colonize (add semicolon)
  • Better Comments (comments with colors)
  • Firebase
  • REST Client (like "Postman" tool)
  • Peacock

Run the following commands in Powershell one line at a time.

 # Flutter
 code --install-extension Dart-Code.flutter

 # Live Share
 code --install-extension ms-vsliveshare.vsliveshare

 # Error Lens
 code --install-extension PhilHindle.errorlens
 
 # Pubspec Assist
 code --install-extension jeroen-meijer.pubspec-assist

 # Colonize
 code --install-extension vmsynkov.colonize

 # Better Comments
 code --install-extension aaron-bond.better-comments

 # Firebase
 code --install-extension toba.vsfire

 # Rest Client
 code --install-extension humao.rest-client

 # Peacock
 code --install-extension johnpapa.vscode-peacock

Upgrade Flutter SDK

In this step onwards, continue working on the new Powershell terminal.

Flutter SDK version from the chocolatey repository might not be the latest. So, upgrade now by running the following comamnd.

flutter upgrade

Diagnos the Fluter Installation

Run the following command to inspect your installation.

 flutter doctor

You may get some problems with your installation. Fix them according the recommendation shown from the previous command.

Fixing Problem: Unable to Locate Android SDK (under Android toolchain - develop for Android Devices)

  • Install Android Studio from: https://developer.android.com/studio/index.html
  • On first launch it will assist you in installing the Android SDK components. (or visit https://flutter.dev/docs/get-started/install/windows#android-setup for detailed instructions).
  • If the Android SDK has been installed to a custom location, please use flutter config --android-sdk to update to that location.

Fixing Problem: Cannot find Chrome

The problem can be fixed by installing Chrome Web Browser.

 # Install Chrome Web Browser
 choco install googlechrome -y

 # To install Edge Web Browser
 #  choco install microsoft-edge -y

To run flutter doctor again, you should open a new powershell session.

  • Web browser (Chrome or Edge) is required if you want to test out your app on web rather than on Android emulators.
  • If you prefer to use Edge browser and already installed it on your computer, flutter doctor may still show you the same problem above. You can simply ignore the problem.

Fixing Problem: Visual Studio not installed

  • Download at https://visualstudio.microsoft.com/downloads/.
  • Please install the "Desktop development with C++" workload, including all of its default components

This is required if you want to test out your app on Windows native. It is fine to simply ignore the above problem since you can test out your app on Android emulators or Web.

Create a New Project

You can create a new project in VS Code. However, in this step we are going to do it using flutter command-line. Run the following commands in the powershell terminal.

 
 # Create a new flutter project
  flutter create project1

 # Open the project into vs code
 code project1

Test Run on Web

  • Here, you are suppose in VS Code now.
  • Select "Chrome" or "Edge" as the device to run on
  • Click on the Run and Debug button and Choose Dart & Flutter as the environment.
