This codelab will walk you through how to setup Visual Studio Code for C++ Programming using the Chocolatey installation manager. You will be installing the followings:

  • MS Visual Studio Code (the source code editor)
  • C/C++ VS Code extension by Microsoft (for C/C++ IntelliSense, debugging, and code browsing). This extension is compulsory.
  • Some other VS Code extensions, such as Colonize, Rainbow Bracket, etc). These are optional.
  • MinGW (Compiling and debugging tools, i.e., gcc / g++ and gdb). It supports both 64-bit and 32-bit addressing.

The installation is only for Windows machine

Chocolatey is a third-party tool. So we need to install it first.

Open a Poweshell Terminal as Administrator

  • Right click the Windows Icon. to open Windows menu.
  • Click on Windows Powershell(Admin)

Install Chocolatey

  • In the Powershell Terminal, run the following commands:
    Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
  • Test chocolatey installation
    choco --version

# If the chocolatey successfully installed, you should see the version.

In this step, we are going to install the following softwares:

  • Visual Studio Code (Code Editor)
  • MingGW (for C++ compiler and debugger)
choco install vscode mingw -y

Open A New Powershell Session

The above installations may have changed the Environement Variables of your system such as the PATH. To allow the changes to take effect, open a new powershell session.

In this step onwards, you are expected to work on the newly opened Powershell terminal. You can install VS Code Extensions from the VS Code. However, in this step we are going to do that using command-line in powershell terminal.

Extenstions to install

  • C/C++
  • Live Share (for collaborative coding)
  • Error Lens (Show error at coding-time)
  • Colonize (add semicolon)
  • Better Comments (comments with colors)

Run the following commands in Powershell one line at a time.

 # C/C++
 code --install-extension ms-vscode.cpptools

 # Live Share
 code --install-extension ms-vsliveshare.vsliveshare

 # Error Lens
 code --install-extension PhilHindle.errorlens
 
 # Colonize
 code --install-extension vmsynkov.colonize

 # Better Comments
 code --install-extension aaron-bond.better-comments

When working with VS Code, you always want to work in a folder. You may have:

  • a folder containing one or more programs. I call this type of program, single-file programs. That means, you write all your code in a single .cpp file for each program. To run such a program, use the Run and Debug: "Console Program".
  • a folder dedicated to a program. This configuration you want to use for your project. You structure your program in a folder and it contains all the .cpp, .hpp. To run this type of program, use the Run and Debug: "Multi-file Console Project".
  • Create a new folder (in Windows File Explorer) . Let's call this folder, program1.
  • Copy vscode setting folder (i.e., .vscode ) from any of your existing project (for example from the provided sample program) to the new folder program1.
  • Open the folder program1 into a VS Code instance.
  • Now, you can proceed adding your source code files .cpp, etc.
