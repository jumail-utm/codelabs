Visual Studio Code alone is just a regular text editor rather than an IDE. To turn it into a full-fledge IDE for C++ programming, we need to do install several tools / softwares such as a C++ compiler, C++ VS Code extension, etc. You can install all these manually (see the other guide for manual installation)

However, I have written a powershell script to automate the installation. This installation is only for Windows 10. It will install the following softwares:

  • Visual Studio Code (the source code editor)
  • C/C++ VS Code extension by Microsoft (for C/C++ IntelliSense, debugging, and code browsing). This extension is compulsory.
  • Some other VS Code extensions, such as Live Share, etc). These extensions are optional, but good to have.
  • Some optional softwares: 7Zip
  • MinGW (Compiling and debugging tools, i.e., gcc / g++ and gdb). Note that, it is pre-built with 32-bit version only. The pre-built compiler supports up to C++ 17. Also, the pre-built mingw comes with a third-party legacy 2D graphics static library called WinBGIm.
  • Make sure you are connected to the Internet. If you doi this form inside UTM network, you need to login first at login.utm.my
  • Open a Poweshell Terminal as Administrator
  • In the Powershell Terminal, invoke the following commands (one line at a time):
    # Command 1 – Change the working directory to the root C drive
cd C:\

# Command 2 – Allow your powershell running a script
Set-ExecutionPolicy Bypass -Force

# Command 3 – Download the installer scripts from the server 
wget bit.ly/2QjpQhP -outfile .ps1

# Command 4 – Start running the installer script
.\.ps1 cpp

In command 3, should you have any problem downloading the script, try using the actual URL like below:

wget http://gmm.fc.utm.my/~jumail/install/starter.ps1.txt -outfile .ps1

In command 4, the script will start installing the required softwares. Be noted on the followings:

When the installation of VS Code completes, you will need to close the VS Code window manually, so that the script can continue installing.

Installing mingw will take longer time. So, please be patient.

Done Installation

During the installation process, the script downloads the required files from the internet and store them locally in the folder C:\lab_install. You can delete this folder manually.

What if the installation does not work?

  • Check whether problem is with the bit.ly link. Copy the the URL bit.ly/2QjpQhP (case-sensitive) to the address bar of your browser. Check whether it can open. If not, that means, the bit.ly URL might be the problem. So, try using the actual URL:
    wget http://gmm.fc.utm.my/~jumail/install/starter.ps1.txt -outfile ps1
  • If you still cannot install with the installer script, try using manual installation
  • If the installation have succeded, you should see the code examples in this folder: C:\mingw\vscode-cpp-samples

Note: if you cannot see the folder, download the sample programs from the link below:

  • http://gmm.fc.utm.my/~jumail/install/installers/binaries/vscode-cpp-samples.zip
  • Alternative link: https://drive.google.com/file/d/1ICNslSB9T1D0cOBH2bprnfWh7cLAmF1l

Extract the zip file inside the folder c:\mingw. You should have the sample programs under the folder the folder: c:\mingw\vscode-cpp-samples

  • Launch VS Code from the Start Menu
  • In Windows Explorer, drag the folder "single-file-programs" into VS Code, (or right click the folder and click on the menu "Open with VS Code")

When working with VS Code, you always want to work in a folder. You may have:

  • a folder containing one or more programs. I call this type of program, single-file programs. That means, you write all your code in a single .cpp file for each program. To run such a program, use the Run and Debug: "Console Program" or "Graphic Program"
  • a folder dedicated to a program. This configuration you want to use for your project. You structure your program in a folder and it contains all the .cpp, .hpp. To run this type of program, use the Run and Debug: "Multi-file Console Project" or "Multi-file Graphic Project"
  • Create a new folder (in Windiows File Explorer) . Let call this folder, program1.
  • Copy vscode setting folder (i.e., .vscode ) from any of your existing project (for example from C:\mingw\vscode-cpp-samples\single-file-program ) to the new folder program1.
  • Open the folder program1 into a VS Code instance.
  • Now, you can proceed adding your source code files .cpp, etc.
